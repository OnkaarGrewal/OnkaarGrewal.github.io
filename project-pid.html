Overview

I built this simulation to actually understand PID control from first principles — not just read the theory. The simulation models a second-order dynamic system and lets me tune the proportional, integral, and derivative gains while watching the system response update in real time.

The simulation uses 4th-order Runge-Kutta integration to solve the differential equations at each timestep. The output is a live Matplotlib window showing the system state, controller output, and error over time — so you can immediately see the effect of gain changes.

This is the first model in what I'm building as a progressive simulation suite — from this inverted pendulum baseline to beam balancing, then drone/quadrotor attitude, and eventually spacecraft attitude control with reaction wheels.

Process & Challenges

The biggest challenge was getting the integrator to not wind up when the system was saturated. I implemented a basic anti-windup scheme where the integrator only accumulates error when the controller output is within the actuator limits.

I also had to think carefully about the discretisation timestep — too large and the RK4 integrator accumulates significant error; too small and the simulation is unnecessarily slow. I settled on a variable timestep approach using SciPy's ODE solver for accuracy.
